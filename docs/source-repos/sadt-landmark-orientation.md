# SADT — Landmark ID (ALI) & Auto Orientation (ASO)

_Area key: `SADT-landmark-orientation`_

## Purpose
Two families of CLI modules from SlicerAutomatedDentalTools (SADT) used to (1) automatically identify anatomical/dental landmarks (ALI = Automatic Landmark Identification) on CBCT volumes and intraoral scans (IOS), and (2) automatically orient/align (ASO = Automatic Slicer Orientation) those volumes/meshes to a reference ("gold") frame. ALI produces 3D landmark fiducials (Slicer .mrk.json, LPS); ASO uses landmarks (or tooth-mean points) to compute a rigid 4x4 transform that re-orients the scan/mesh. The landmark-driven orientation is almost entirely classical geometry (vector angle/axis, Rodrigues rotation matrices, ICP) and ports directly to TypeScript; the landmark *detection* nets are heavier (deep-RL agents on CBCT, multi-view UNet on IOS) and depend on downloaded weights.

**Tech stack:** Python, PyTorch, PyTorch Lightning (ASO_CBCT Net only), MONAI (DenseNet 3D, UNet 2D, transforms: EnsureChannelFirst/BorderPad/SpatialCrop/ScaleIntensity/AsDiscrete), PyTorch3D (FoVPerspectiveCameras, MeshRasterizer, HardPhongShader, look_at_rotation — ALI_IOS), SimpleITK (read/resample/transforms: TranslationTransform, VersorRigid3DTransform, Euler3DTransform, CompositeTransform, AffineTransform), ITK (ResampleImageFilter, interpolators — ALI_CBCT SetSpacing), VTK (polydata IO, vtkIterativeClosestPointTransform, normals, octree locator, transform filters), NumPy / SciPy, dicom2nifti + GDCM (DICOM import), 3D Slicer CLI module framework (.xml descriptors, slicer.util.pip_install)

**License:** Slicer-style / SADT repository license (top-level LICENCE file in SlicerAutomatedDentalTools-main; per-module .xml headers say <license>Slicer</license>). Effectively permissive open-source tied to the 3D Slicer ecosystem; confirm exact terms in the repo LICENCE before redistribution.

**I/O formats:** CBCT volumes: .nii/.nii.gz, .nrrd/.nrrd.gz, .gipl/.gipl.gz, DICOM series in (converted to NIfTI via dicom2nifti/GDCM), Intraoral/surface meshes: .vtk, .vtp, .stl, .off, .obj (with per-vertex segmentation arrays Universal_ID / PredictedID for IOS), Landmarks/fiducials: 3D Slicer Markups JSON (.mrk.json / .json, LPS coordinate system), Transforms out: SimpleITK .tfm (rigid/affine) and raw 4x4 .npy matrices, Resampled/oriented volumes out: .nii.gz; oriented meshes out: .vtk/.vtp/.obj

## Modules

### ALI_CBCT (landmark ID on CBCT)
Multi-agent deep reinforcement-learning landmark finder. Each landmark gets one 'agent' that walks a discrete 6-neighbour grid (Up/Down/Back/Front/Left/Right) over a multi-scale CBCT. A per-landmark 'Brain' (one network per scale) predicts the next move from a cropped FOV around the agent; the agent climbs from coarse (1mm) to fine (0.3mm) spacing, stops when it revisits a voxel (short-memory), then a 6-direction 'Focus' averages the final position. Output written as grouped Slicer fiducial JSON (groups CB/U/L/CI). Preprocessing: histogram percentile clamp + multi-scale resampling.

- **Algorithm:** Deep-RL voxel-walking agents; network = MONAI DenseNet 3D feature extractor (growth_rate=34, block_config (6,12,24,16), out=1024 feats) -> small 4-layer FC head (1024->512->256->128->6 moves), argmax action. One network per scale per landmark.
- **Entry points:** `ALI_CBCT/ALI_CBCT.py`, `ALI_CBCT/ALI_CBCT_utils/brain.py`, `ALI_CBCT/ALI_CBCT_utils/agent.py`, `ALI_CBCT/ALI_CBCT_utils/environment.py`, `ALI_CBCT/ALI_CBCT_utils/preprocess.py`, `ALI_CBCT/ALI_CBCT_utils/constants.py`

### ALI_IOS (landmark ID on intraoral surface scans)
Landmark detection on segmented tooth surface meshes (.vtk with per-vertex region IDs). For each requested tooth/landmark type, the mesh is centered+scaled to unit, a PyTorch3D camera orbits the tooth from several viewpoints, renders RGB(normals)+depth images, and a 2D UNet segments which pixels are the landmark (R/G/B channels = O/MB/DB or CL/CB). Predicted pixels are back-projected to mesh faces via the rasterizer's pix_to_face buffer, faces filtered to the target tooth, vertex positions averaged, snapped to nearest surface point (vtkOctreePointLocator), and up-scaled back to world coords.

- **Algorithm:** Multi-view differentiable rendering (PyTorch3D look_at_rotation cameras + MeshRasterizer/HardPhongShader) feeding a MONAI 2D UNet (spatial_dims=2, in=4 [RGB+depth], out=4, channels (16,32,64,128,256,512), num_res_units=4); landmark recovered by pixel->face->vertex back-projection (heatmap-style, argmax onehot).
- **Entry points:** `ALI_IOS/ALI_IOS.py`, `ALI_IOS/ALI_IOS_utils/surface.py`, `ALI_IOS/ALI_IOS_utils/agent.py`, `ALI_IOS/ALI_IOS_utils/render.py`, `ALI_IOS/ALI_IOS_utils/model.py`, `ALI_IOS/ALI_IOS_utils/io.py`

### ASO_CBCT / SEMI (landmark-based CBCT orientation)
The portable orientation path. Loads input landmarks (source) and gold-standard landmarks (target) from Slicer JSON, optionally drops mis-located landmarks by comparing pairwise inter-landmark distances and direction angles to gold, runs an analytic 'InitICP' (1 translation + 2 axis-angle rotations chosen from the 3 landmarks that minimize mean distance via FindOptimalLandmarks brute force), then a VTK rigid-body ICP refinement. Builds a composite SimpleITK transform, resamples the CBCT, and writes the oriented scan + transform + transformed landmarks.

- **Algorithm:** Closed-form rigid alignment from landmark correspondences: translation to match first landmark, axis-angle (AngleAndAxisVectors + Rodrigues RotationMatrix) rotations, brute-force best-triple selection, then vtkIterativeClosestPointTransform (RigidBody, StartByMatchingCentroids). Pure linear algebra + VTK/SITK; no neural net at inference.
- **Entry points:** `ASO_CBCT/SEMI_ASO_CBCT/SEMI_ASO_CBCT.py`, `ASO_CBCT/ASO_CBCT_utils/utils.py`

### ASO_CBCT / PRE (DL pre-orientation, partially wired)
Intended deep-learning coarse pre-orientation. In THIS checkout PRE_ASO_CBCT.py only centers the volume (translates origin to image center) and writes a translation .tfm; the actual direction-regression DenseNet169 (Net.py) is present but commented out of the package __init__ and not invoked by the PRE entry script. So the DL pre-orientation is effectively a stub here and the real work is done by SEMI (landmark ICP).

- **Algorithm:** DenseNet169 (MONAI, spatial_dims=3, in=1, out=3) regressing a unit direction vector trained with 1 - cosine-similarity loss (PyTorch Lightning) — defined but not loaded/executed in the shipped PRE pipeline; runtime PRE = volume centering only.
- **Entry points:** `ASO_CBCT/PRE_ASO_CBCT/PRE_ASO_CBCT.py`, `ASO_CBCT/ASO_CBCT_utils/Net.py`, `ASO_CBCT/ASO_CBCT_utils/ResamplePreASO.py`

### ASO_IOS / PRE + SEMI (intraoral mesh orientation)
Orients tooth surface meshes to a gold reference. PRE ('PrePreAso') uses 3-4 tooth-mean points to build a left/middle/right plane and analytically rotates the source plane normal+direction onto the target plane, then translates centroids — a deterministic, weight-free coarse alignment. SEMI then runs a composable ICP chain (analytic InitIcp from best landmark triple, then VTK rigid ICP) using selected landmarks, transforms the surface + landmark JSON, supports occlusion 'linking' so the opposite jaw inherits the same matrix. Tooth-mean extraction reads per-vertex 'Universal_ID' segmentation.

- **Algorithm:** Tooth-centroid plane alignment (cross-product normals + Rodrigues rotations, PrePreAso) followed by InitIcp (landmark-triple axis-angle init) + vtkIterativeClosestPointTransform rigid refinement; transforms composed as 4x4 matrices. Fully classical geometry — the highest-value port target.
- **Entry points:** `ASO_IOS/PRE_ASO_IOS/PRE_ASO_IOS.py`, `ASO_IOS/SEMI_ASO_IOS/SEMI_ASO_IOS.py`, `ASO_IOS/ASO_IOS_utils/icp.py`, `ASO_IOS/ASO_IOS_utils/pre_icp.py`, `ASO_IOS/ASO_IOS_utils/transformation.py`, `ASO_IOS/ASO_IOS_utils/utils.py`

## Reusable utilities

- `ASO_CBCT/ASO_CBCT_utils/utils.py` — Self-contained rigid-alignment math (NO torch): RotationMatrix(axis,theta) Rodrigues; AngleAndAxisVectors(v1,v2); InitICP analytic landmark init; FindOptimalLandmarks brute-force best-triple; ApplyTransform/ApplyTranslation on landmark dicts; ComputeMeanDistance; GetDistDifference/GetDirDifference + GetLandmarkToRemove outlier rejection; ConvertToVTKPoints; VTKMatrixToNumpy; LoadJsonLandmarks/WriteJson (Slicer Markups schema). Prime TS port target.
- `ASO_IOS/ASO_IOS_utils/pre_icp.py` — PrePreAso(): deterministic 3/4-tooth plane alignment using cross-product normals and two Rodrigues rotations + centroid translation, returns a 4x4 matrix. make_vector() builds plane normal+direction from 3 points. organizeLandmark() left/middle/right ordering. Pure numpy — directly portable.
- `ASO_IOS/ASO_IOS_utils/transformation.py` — Matrix/transform helpers (numpy): RotationMatrix (Rodrigues), TransformDict (apply 4x4 to landmark dict), TransformList (apply 4x4 to Nx3 array), TranslationDict, ApplyTransform dispatcher. Zero ML deps.
- `ASO_IOS/ASO_IOS_utils/icp.py` — Composable ICP framework: InitIcp (analytic landmark-triple init with caching), vtkICP wrapper, vtkMeanTeeth/vtkMiddleTeeth/vtkIterTeeth (extract per-tooth centroid points from segmented mesh by Universal_ID), SameNumberPoint resampling helpers, DictTovtkPoints, VTKMatrixToNumpy. The numpy InitIcp/AngleAndAxisVectors/ComputeMeanDistance subset is ML-free and portable.
- `ALI_CBCT/ALI_CBCT_utils/preprocess.py` — CorrectHisto() percentile histogram clamp (cumulative-histogram based intensity windowing); SetSpacing()/ResampleImage() isotropic resampling with origin re-centering math; convertdicom2nifti() (GDCM series reader w/ dicom2nifti fallback). Resampling/origin math is reusable; intensity windowing logic ports to TS.
- `ALI_CBCT/ALI_CBCT_utils/constants.py` — GROUP_LABELS/LABELS full dental+craniofacial landmark taxonomy (CB/U/L/CI groups, ~200 named landmarks) and the 6-direction MOVEMENT_MATRIX — useful reference data for any landmark UI.
- `ALI_IOS/ALI_IOS_utils/surface.py` — Mesh ops (VTK+numpy): ScaleSurf() center+unit-scale by bbox with reversible mean/scale_factor; Upscale() inverse; ComputeNormals; GetColorArray (encode normals as RGB); GetSurfProp (verts/faces/normals/region_id tensors); RemoveExtraFaces (filter faces to a tooth label). ScaleSurf/Upscale normalization math is portable.
- `ALI_IOS/ALI_IOS_utils/model.py` — Pure lookup tables: tooth->FDI numbering, per-tooth landmark name maps (dic_label), multi-view camera direction sets (dic_cam) for occlusal vs buccal models, RGB-channel->landmark-type map (MODELS_DICT). No ML; reusable config for an IOS landmark UI.
- `ASO_CBCT/ASO_CBCT_utils/ResamplePreASO.py` — Standalone SimpleITK resampler resample_fn/Resample/PreASOResample: fit-spacing, iso-spacing, fixed-size (128^3) with center/origin recomputation and identity-direction option. Clean reference for volume normalization.
- `ALI_IOS/ALI_IOS_utils/io.py & ASO_CBCT_utils/utils.py (WriteJson/GenControlPoint)` — Builders/parsers for 3D Slicer Markups fiducial JSON (controlPoints, LPS, full display block). Needed to read/write landmark files interoperable with Slicer; trivially portable to TS JSON.

- **ASO rigid-alignment math core (RotationMatrix/Rodrigues, AngleAndAxisVectors, ApplyTransform/ApplyTranslation, ComputeMeanDistance, VTKMatrixToNumpy)** — The self-contained linear-algebra kernel from ASO_CBCT_utils/utils.py and ASO_IOS_utils/transformation.py: axis-angle->4x4 rotation (Rodrigues), angle/axis between two vectors, apply a 4x4 to an Nx3 landmark set or landmark dict, mean inter-point distance. Zero ML/VTK/torch dependency. Foundational dependency for every other ASO item below. CBCTer has three.js loaded, so most primitives exist; only the landmark-dict apply helpers are new. This is the highest value-to-effort win in the whole area.
- **Slicer Markups fiducial JSON I/O (LoadJsonLandmarks / WriteJson / GenControlPoint)** — Reader/writer for 3D Slicer .mrk.json landmark files (controlPoints array, LPS coordinate system, display block) from ASO_CBCT_utils/utils.py and ALI_IOS_utils/io.py. Also the landmark group/label taxonomy in ALI_CBCT constants.py (CB/U/L/CI groups, ~200 named craniofacial/dental landmarks). CBCTer has NO landmark/fiducial concept today (confirmed: no landmark/Markups types in src/domain). This unlocks interop with the entire Slicer/SADT ecosystem and is a prerequisite for any landmark-driven UI or orientation. Watch LPS-vs-voxel/world coordinate conversion against CBCTer's volume frame.
- **ASO_CBCT SEMI landmark-based CBCT orientation (InitICP + outlier rejection + rigid ICP)** — The portable orientation pipeline: analytic InitICP (translation + 2 axis-angle rotations from the best landmark triple via FindOptimalLandmarks brute force), GetLandmarkToRemove outlier rejection (compare pairwise distance/direction diffs to gold), then a VTK rigid-body ICP refinement; produces a 4x4 transform that re-orients the CBCT to a gold frame. _(depends on: ASO rigid-alignment math core (RotationMatrix/Rodrigues, AngleAndAxisVectors, ApplyTransform/ApplyTranslation, ComputeMeanDistance, VTKMatrixToNumpy), Slicer Markups fiducial JSON I/O (LoadJsonLandmarks / WriteJson / GenControlPoint))_ No neural net at inference — pure geometry, the explicitly flagged prime port target. Main effort is a baked voxel-resample worker (reuse roi.ts indexing patterns) if you want an oriented volume rather than just a display transform. Needs a stored gold/reference landmark set (the only ASO 'model' artifact is a fixed reference transform/landmarks).
- **ASO_IOS PrePreAso deterministic tooth-plane coarse alignment** — Weight-free coarse mesh orientation (ASO_IOS_utils/pre_icp.py): from 3-4 tooth-mean points build a left/middle/right plane (cross-product normal + direction), analytically rotate the source plane onto the target plane with two Rodrigues rotations, then translate centroids. Returns a 4x4. _(depends on: ASO rigid-alignment math core (RotationMatrix/Rodrigues, AngleAndAxisVectors, ApplyTransform/ApplyTranslation, ComputeMeanDistance, VTKMatrixToNumpy))_ Value depends on whether CBCTer targets intraoral-scan (IOS) meshes — currently it is CBCT-centric. High value if IOS import is on the roadmap; otherwise medium. Depends on tooth-centroid extraction (next item) to get the input points.
- **Per-tooth centroid extraction from segmented mesh (vtkMeanTeeth / vtkMiddleTeeth by Universal_ID)** — From a mesh/labelmap with per-vertex tooth IDs (Universal_ID / PredictedID), compute each tooth's mean point — the input points consumed by PrePreAso and InitICP for IOS orientation. From ASO_IOS_utils/icp.py. CBCTer already splits teeth (watershed + per-tooth masks in src/lib/segmentation/generateLibrary.ts), so centroids are nearly free from existing data. Feeds the IOS orientation and could also drive an arch-coordinate / FDI-numbering UI.
- **ALI_CBCT deep-RL landmark agents (per-landmark DenseNet3D 'Brain')** — Multi-agent reinforcement-learning landmark finder: one agent per landmark walks a 6-neighbour voxel grid over a multi-scale CBCT; a MONAI DenseNet3D feature extractor + 4-layer FC head predicts the next move (6 logits, argmax); coarse->fine scale climb with short-memory stop and 6-direction focus averaging. _(depends on: ASO rigid-alignment math core (RotationMatrix/Rodrigues, AngleAndAxisVectors, ApplyTransform/ApplyTranslation, ComputeMeanDistance, VTKMatrixToNumpy), Slicer Markups fiducial JSON I/O (LoadJsonLandmarks / WriteJson / GenControlPoint))_ Weights are NOT on disk (Slicer downloads per landmark per scale on demand) — large multi-model download/caching story (potentially dozens of .onnx files). Sequential per-step inference (one forward pass per agent step, many steps) is slow in WASM; this is the costly path. Value is real (auto-landmarking enables auto-orientation end-to-end) but effort + model-acquisition risk are high. Could alternatively live as a server-side microservice if in-browser latency is unacceptable.
- **ALI_IOS multi-view UNet landmark detection on surface meshes** — Landmark detection on intraoral tooth meshes via differentiable rendering: PyTorch3D cameras orbit a tooth, render RGB(normals)+depth, a MONAI 2D UNet (in=4,out=4) segments landmark pixels, which are back-projected to faces/vertices via the rasterizer pix_to_face buffer and averaged to a surface point. _(depends on: Slicer Markups fiducial JSON I/O (LoadJsonLandmarks / WriteJson / GenControlPoint), Per-tooth centroid extraction from segmented mesh (vtkMeanTeeth / vtkMiddleTeeth by Universal_ID))_ Requires IOS surface-scan input that CBCTer does not currently ingest, plus a from-scratch three.js reimplementation of PyTorch3D's rasterizer back-projection (the genuinely hard, error-prone part). Weights also not on disk. Lowest priority unless IOS becomes a first-class input.
- **ASO_CBCT PRE DL direction-regression pre-orientation (DenseNet169)** — Intended coarse DL pre-orientation: a MONAI DenseNet169 (3D, out=3) regressing a unit direction vector (1 - cosine-similarity loss). In this checkout it is COMMENTED OUT of the package __init__ and never invoked — runtime PRE only centers the volume. Effectively a stub in this checkout. Skip the DenseNet169; if a centering step is ever wanted, add a 3-line origin-recenter using existing volume meta. Listed only to record that it was assessed and intentionally dropped.

## Already covered by CBCTer (skip)

- CBCT volume loading + isotropic resampling/origin-recenter math (ALI_CBCT preprocess SetSpacing/ResampleImage, ASO_CBCT ResamplePreASO) is already covered by CBCTer's import/volume pipeline and ROI crop (extractCropFloat32 in src/lib/segmentation/roi.ts) — re-port only the bits ASO/ALI consumes, not a new resampler.
- Intensity windowing/percentile clamp (ALI_CBCT CorrectHisto) overlaps CBCTer's window/level math in src/lib/volume/math.ts and the toothSeg worker's NormalizeIntensity; no need to re-port histogram windowing as a standalone capability.
- Mask-to-surface meshing + STL/PLY export (ALI_IOS surface mesh handling, VTK polydata IO) is already done in src/lib/segmentation/maskMesh.ts (maskToBinaryStl/maskToAsciiPly) and src/lib/surface/generateSurface.ts.
- onnxruntime-web sliding-window 3D inference plumbing already exists (src/workers/toothSeg.worker.ts) — any ALI net port reuses this worker pattern, it is not new infrastructure.
- DICOM->volume import incl. ImageOrientationPatient handling (ALI_CBCT convertdicom2nifti/GDCM) is covered by src/lib/import/adapters/dicom/* and @itk-wasm/dicom.
- Sigmoid/argmax heatmap->mask thresholding and connected-components/keep-largest cleanup (used downstream of ALI nets) already exist in src/lib/segmentation/maskOperations.ts and connectedComponents.ts.
