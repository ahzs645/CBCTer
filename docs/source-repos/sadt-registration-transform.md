# SADT — Registration & Transforms (AREG / AutoMatrix / AutoCrop3D / FlexReg)

_Area key: `SADT-registration-transform`_

## Purpose
This area of SlicerAutomatedDentalTools (SADT) handles automatic registration and transform application for dental imaging: CBCT volume-to-volume registration, intra-oral scan (IOS) mesh-to-mesh registration, cross-modality CBCT-to-IOS registration, batch application of transform matrices to volumes/meshes/landmarks, and ROI cropping of volumes. Registration is a MIX of classical (ITK/Elastix voxel registration, VTK/SciPy ICP, landmark transforms) and deep learning: the only learned component is a 2D MONAI U-Net (rendered multi-view mesh segmentation, PyTorch/PyTorch3D) used in AREG_IOS to predict a 'butterfly' palate patch that drives mesh ICP. The actual spatial alignment is always classical (Elastix rigid for CBCT, VTK/SciPy ICP for meshes). All modules are 3D Slicer scripted modules: a UI module wraps a CLI (named *_CLI or *_Method) that does the real work; the registration/transform/crop math is well isolated in the CLI utils and is highly portable.

**Tech stack:** Python, 3D Slicer (scripted modules + CLI), SimpleITK (sitk), ITK + ITK-Elastix (itk.ElastixRegistrationMethod), VTK (vtkIterativeClosestPointTransform, vtkLandmarkTransform, vtkTransformPolyDataFilter, polydata IO), PyVista, PyTorch, PyTorch Lightning, MONAI (UNet, DiceCELoss), PyTorch3D (mesh rendering / rasterization), torchmetrics, NumPy, SciPy (cKDTree, SVD/Kabsch ICP), dicom2nifti, GDCM (via SimpleITK ImageSeriesReader)

**License:** No LICENSE file found inside any of the 9 inspected module directories; SlicerAutomatedDentalTools is published under Apache-2.0 at the repo root (root not inspected here, so unconfirmed in these folders).

**I/O formats:** CBCT volumes: NIfTI (.nii, .nii.gz), NRRD (.nrrd), GIPL (.gipl) ; DICOM input converted to NIfTI, Meshes/surfaces: VTK legacy (.vtk), VTP (.vtp), STL (.stl), OBJ (.obj, with optional .mtl), OFF (.off), Transforms IN: SimpleITK .tfm, ITK HDF5 .h5, MATLAB .mat, NumPy .npy, plain-text .txt (AutoMatrix); internally 4x4 numpy / VTK vtkMatrix4x4 / sitk Euler3D & Affine, Transforms OUT: SimpleITK .tfm (Euler3DTransform for CBCT, AffineTransform for IOS/FlexReg), Landmarks: Slicer markups JSON (.mrk.json / .json, markups[0].controlPoints[].position), ROI: Slicer markups ROI JSON (.mrk.json with center + size)

## Modules

### AREG_CBCT (CBCT-to-CBCT voxel registration)
Automatic rigid registration of two CBCT volumes (T1 fixed, T2 moving) using ITK-Elastix. The fixed image is masked by a segmentation label (CB/MAND/MAX -> cranial base/mandible/maxilla), then itk.ElastixRegistrationMethod runs a deterministic multi-resolution rigid EulerTransform (AdvancedMattesMutualInformation, ConjugateGradient, 3-level pyramid). The Elastix parameter map is converted to a SimpleITK Euler3DTransform, composed, and used to resample the moving image. Outputs a .tfm transform and a resampled .nii.gz. NOT deep-learning; pure ITK -> itk-wasm/Elastix-wasm candidate. Also bundles DICOM->NIfTI conversion (SimpleITK GDCM with dicom2nifti fallback).

- **Algorithm:** ITK-Elastix rigid registration (EulerTransform, Mattes MI) + SimpleITK resampling; NO neural net
- **Entry points:** `AREG_CBCT/AREG_CBCT.py`, `AREG_CBCT/AREG_CBCT_utils/utils.py`

### AREG_IOS (intra-oral scan mesh registration)
Registers two IOS surface meshes (T1/T2 .vtk). A 2D MONAI U-Net (MonaiUNetHRes) renders the mesh from 7 camera viewpoints with PyTorch3D, segments a 'Butterfly' palate patch per vertex, post-processes it (RemoveIslands/Dilate/Erode), then registers T2->T1 via VTK ICP (vtkIterativeClosestPointTransform, rigid) restricted to the Butterfly-labeled vertices. The 4x4 ICP matrix (numpy) is also exported as a SimpleITK AffineTransform .tfm (optionally composed with an ASO orientation matrix). Lower-arch meshes are co-registered by re-using the upper-arch matrix.

- **Algorithm:** MONAI 2D U-Net multi-view mesh segmentation (PyTorch/PyTorch3D) -> VTK rigid ICP on labeled region; transform math in numpy + SimpleITK
- **Entry points:** `AREG_IOS/AREG_IOS.py`, `AREG_IOS/AREG_IOS_utils/net.py`, `AREG_IOS/AREG_IOS_utils/PredPatch.py`, `AREG_IOS/AREG_IOS_utils/ICP.py`, `AREG_IOS/AREG_IOS_utils/transformation.py`

### AREG_IOSCBCT (cross-modality CBCT<->IOS registration)
Registers an IOS mesh to a CBCT-derived mesh/landmarks for upper and lower jaws. Coarse alignment via vtkLandmarkTransform (rigid, from matched landmark sets), then a self-contained point-to-plane ICP implemented from scratch with scipy.spatial.cKDTree (nearest-neighbor correspondence) + SVD-based rigid solve (Kabsch). Explicitly avoids Open3D ('no Open3D dependency'). Transforms applied with PyVista/numpy; landmarks transformed by 4x4 matrix multiply. Uses SimpleITK only to read the CBCT scan.

- **Algorithm:** vtkLandmarkTransform rigid init + custom SciPy-KDTree/SVD point-to-plane ICP; NO neural net, NO Open3D
- **Entry points:** `AREG_IOSCBCT/AREG_IOSCBCT.py`

### AREG (dispatcher / Slicer UI front-end)
Top-level Slicer scripted module that selects a registration Method (CBCT / IOS / IOSCBCT) and orchestrates the corresponding CLI. Holds model/reference/test-data download URLs (GitHub releases HUTIN1/AREG, HUTIN1/ASO) and a robust torch/torchvision/PyTorch3D installer. No registration math itself.

- **Algorithm:** Orchestration + model/data download + dependency install (no algorithm)
- **Entry points:** `AREG/AREG.py`, `AREG/AREG_Method/Method.py`, `AREG/AREG_Method/CBCT.py`, `AREG/AREG_Method/IOS.py`, `AREG/AREG_Method/IOSCBCT.py`, `AREG/AREG_Method/install_pytorch.py`

### AutoMatrix / Automatrix_CLI (batch transform application)
Applies transform matrices to batches of volumes, meshes, and landmarks. Matches scans to matrices by patient-ID name parsing. For volumes: SimpleITK ResampleImageFilter (linear for images, nearest-neighbor for segmentations, with optional reference image / CompositeTransform handling). For landmarks (.mrk.json): applies the INVERSE sitk transform to each control point. For meshes: VTK transform via the Method. Accepts matrices in .tfm, .h5, .mat, .npy, .txt; scans in .nii/.nii.gz/.nrrd, .vtk/.vtp/.stl/.off/.obj, .mrk.json. Has special 'fromAreg' mode mapping AREG output folder structure (CB/MAX/MAND _OutReg).

- **Algorithm:** SimpleITK resampling + VTK polydata transform + landmark matrix multiply; reads many matrix formats
- **Entry points:** `Automatrix_CLI/Automatrix_CLI.py`, `AutoMatrix/AutoMatrix_Method/applyMatrix.py`, `AutoMatrix/AutoMatrix_Method/General_tools.py`

### AutoCrop3D (ROI volume cropping)
Crops volumes (.nii/.nii.gz/.nrrd/.gipl) to a Slicer markups ROI (.mrk.json with center+size). Converts ROI physical bounds to image indices via TransformPhysicalPointToContinuousIndex, clamps to image bounds, and either returns the cropped sub-volume or re-embeds it into a blank full-size image ('box_Size' option). Supports a folder of ROIs matched per-patient. Can emit a VTK surface from cropped segmentations.

- **Algorithm:** SimpleITK index-space ROI crop driven by Slicer markups ROI JSON; classical CV, no learning
- **Entry points:** `AutoCrop3D/Crop_Volumes_CLI/AutoCrop3D_CLI.py`, `AutoCrop3D/Crop_Volumes_CLI/Crop_Volumes_utils/FilesType.py`, `AutoCrop3D/Crop_Volumes_CLI/Crop_Volumes_utils/GenerateVTKfromSeg.py`

### FlexReg / FlexReg_CLI (flexible region mesh registration)
Interactive/semi-automatic mesh registration where the user defines a custom registration region ('butterfly' patch, freehand 'curve' patch, or delete) on a tooth-segmented IOS mesh, then registers via VTK rigid ICP restricted to that patch. The butterfly patch is built purely geometrically (tooth centroids from Universal_ID segmentation + Bezier/segment curves + geodesic Dilation flood-fill) — NO neural net here (unlike AREG_IOS). The ICP 4x4 matrix is exported as a SimpleITK AffineTransform .tfm with LPS<->RAS flip composition. torch is used only for tensor/CUDA math, not for any learned model.

- **Algorithm:** Geometric patch construction (centroids, Bezier, mesh dilation) + VTK rigid ICP on patch; torch used as array lib only, no model
- **Entry points:** `FlexReg_CLI/FlexReg_CLI.py`, `FlexReg_CLI/FlexReg_Method/make_butterfly.py`, `FlexReg_CLI/FlexReg_Method/draw.py`, `FlexReg_CLI/FlexReg_Method/ICP.py`, `FlexReg_CLI/FlexReg_Method/propagation.py`, `FlexReg_CLI/FlexReg_Method/orientation.py`

## Reusable utilities

- `AREG_CBCT/AREG_CBCT_utils/utils.py` — MatrixRetrieval (Elastix param map -> sitk Euler3D/Affine), ComputeFinalMatrix (compose list of rotations/translations into one Euler3DTransform), ResampleImage (sitk linear resample), applyMask (apply seg label as mask), make_rigid_param_map_deterministic + ElastixReg (self-contained ITK-Elastix rigid registration), convertdicom2nifti (DICOM->NIfTI with fallback), LoadOnlyLandmarks (markups JSON reader). Highly reusable; the registration core is ~150 lines.
- `AREG_IOS/AREG_IOS_utils/transformation.py` — read_matrix (.tfm -> 4x4 numpy), saveMatrixAsTfm (4x4 numpy -> sitk AffineTransform .tfm with ASO composition + inversion), TransformSurf/RotateTransform (apply 4x4 to vtkPolyData), TransformList/TransformDict (apply 4x4 to point arrays / landmark dict), RotationMatrix (axis-angle -> 3x3), ScaleSurf (center+normalize mesh). Pure numpy/VTK transform math, no deps on the model — top port candidate.
- `AREG_IOS/AREG_IOS_utils/ICP.py` — vtkICP (rigid ICP via vtkIterativeClosestPointTransform returning 4x4 numpy) and ICP orchestrator class that chains ICP steps, composes matrices, and applies to mesh/landmarks/dict. Self-contained mesh registration.
- `AREG_IOS/AREG_IOS_utils/utils.py` — ReadSurf (multi-format mesh reader vtk/vtp/stl/obj), WriteSurf, ComputeNormals, GetColorArray (normals->RGB), LoadJsonLandmarks, VTKMatrixToNumpy (vtkMatrix4x4 -> numpy). Generic mesh/landmark IO helpers.
- `AREG_IOSCBCT/AREG_IOSCBCT.py` — run_icp_point_to_plane (dependency-light point-to-plane ICP using scipy cKDTree + SVD/Kabsch, outlier rejection by max_dist, RMSE/fitness convergence) and align_by_landmarks (vtkLandmarkTransform rigid). Excellent reference implementation of ICP without Open3D — directly portable to numpy/JS.
- `Automatrix_CLI/Automatrix_CLI.py` — ResampleImage / apply_transform_to_image (sitk resample with reference + seg-aware interpolator + CompositeTransform support), apply_transform_to_landmarks (inverse-transform markups JSON control points), GetPatients (scan<->matrix pairing by filename). The canonical 'apply a saved transform to a batch' logic.
- `AutoCrop3D/Crop_Volumes_CLI/AutoCrop3D_CLI.py` — ROI-to-index crop: ROI center/size (physical) -> TransformPhysicalPointToContinuousIndex -> clamp -> sitk slice crop, with optional re-embed into full-size blank image. Compact, self-contained volume cropping routine.
- `FlexReg_CLI/FlexReg_Method/propagation.py` — Dilation / GetNeighbors / Neighbours — geodesic flood-fill region growing over a triangle mesh (vtk GetPointCells/GetCellPoints + torch set ops). Reusable mesh region-grow / patch-closing utility.
- `FlexReg_CLI/FlexReg_Method/vtkSegTeeth.py` — vtkMeanTeeth (centroid of each tooth label), vtkMeshTeeth (build point-cloud polydata from selected labels for ICP), vtkIterTeeth, Universal_ID<->tooth-name (FDI/UNS) mapping. Segmentation-label-driven mesh selection utilities shared by FlexReg and AREG_IOS.
- `FlexReg_CLI/FlexReg_Method/orientation.py` — orientation() — rigid pre-alignment of a mesh to a canonical frame from 3 tooth landmarks (two rotations via axis-angle RotationMatrix + translation), returns transformed surface. Reusable landmark-based mesh orientation.
- `FlexReg_CLI/FlexReg_Method/make_butterfly.py` — butterflyPatch + Bezier_bled + Segment2D — fully geometric construction of the palate registration patch from tooth centroids using Bezier curves, line segments, distance thresholding (torch.cdist) and Dilation. Pure geometry, no model.

- ROI/crop bounds in voxel space — CBCTer already has src/lib/segmentation/roi.ts (clampRoi, extractCropFloat32) and crop clipping in the 3D viewer; AutoCrop3D's voxel-index crop is the same idea, only the physical-ROI→index mapping and re-embed option are new
- Surface mesh generation from masks to STL/PLY with area/volume — CBCTer already has src/lib/surface/generateSurface.ts, src/workers/surface.worker.ts, src/lib/segmentation/maskMesh.ts; SADT only consumes/transforms meshes, it does not generate them differently
- DICOM→volume loading — CBCTer already ingests DICOM via @itk-wasm/dicom and a custom parser; SADT's DICOM→NIfTI (SimpleITK GDCM / dicom2nifti) is just a format bridge CBCTer does not need (it loads into its own LoadedVolume)
- 3D mesh viewing — CBCTer renders meshes/STL in three.js (ToothMeshViewport, ToothArchViewport); SADT has no viewer of its own, only Slicer
- Tooth-label-driven selection plumbing already exists in CBCTer's tooth library pipeline (generateLibrary, watershed split, manifest), overlapping FlexReg/AREG_IOS's reliance on a tooth segmentation upstream
