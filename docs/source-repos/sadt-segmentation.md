# SADT — Segmentation (AMASSS / DentalSeg batch / CLIC / CNE)

_Area key: `SADT-segmentation`_

## Purpose
This area of SlicerAutomatedDentalTools (SADT) covers automatic CBCT/CT skull and dental structure segmentation. The primary, actively-maintained pipelines are AMASSS_CLI (multi-structure skull segmentation: mandible, maxilla, cranial base, upper airway, cervical vertebra, skin, etc.) and BATCHDENTALSEG / DentalSegmentator (jaw + teeth + mandibular canal), both built on nnU-Net v2 3d_fullres PlainConvUNet models with CT-intensity normalization. CLIC is a separate Mask-R-CNN (torchvision) 2D slice-wise segmenter for impacted-canine bone-coverage classification. CNE/CNE_CLI are NOT image segmentation (they extract structured data from clinical text notes via a local GGUF LLM) and are included only to note they are out of scope. AMASSS and CNE top-level folders are thin Slicer GUI module wrappers around their *_CLI counterparts. No model weight files exist on disk in any folder — every model is downloaded at runtime from GitHub releases; only nnU-Net config JSONs are shipped.

**Tech stack:** Python 3.9+, nnU-Net v2 (nnunetv2, nnUNetv2_predict CLI), PyTorch, torchvision (Mask R-CNN ResNet50-FPN), SimpleITK, ITK (itk), cc3d (connected-components-3d), VTK, nibabel, dicom2nifti, numpy (<2.0 pinned), 3D Slicer (slicer, qt, ctk, MRML), SlicerNNUNetLib, SlicerConda / CondaSetUp, llama-cpp-python (CNE only, GGUF), PyGithub (release/weight download)

**License:** Apache License 2.0 (repo-level LICENSE; AMASSS algorithm credited to AMASSS_CBCT, DentalSegmentator weights from gaudot/SlicerDentalSegmentator releases)

**I/O formats:** Input: NIfTI (.nii/.nii.gz), Input: NRRD (.nrrd/.nrrd.gz), Input: MHA/MHD (CLIC), Input: GIPL/.gipl.gz (BATCHDENTALSEG), Input: DICOM series (AMASSS via dicom2nifti, flag isDCMInput), Output: NIfTI/NRRD labelmaps (int16/uint16), Output: VTK polydata meshes (per-label and merged, with RAS->LPS transform), Output: STL, OBJ, glTF (BATCHDENTALSEG via Slicer + SlicerOpenAnatomy), Output (CNE): plain-text field extractions

## Modules

### AMASSS_CLI (nnU-Net v2 multi-structure CBCT segmentation)
Headless CLI that segments multiple skull/dental structures from a CBCT volume (or a folder of volumes). For each requested structure it discovers a nnU-Net v2 model folder, runs inference, then post-processes each binary mask (morphological close + largest-connected-component cleanup), optionally merges all structures into one labelmap by a fixed priority order, writes NIfTI/NRRD labelmaps, and optionally generates VTK surface meshes via DiscreteMarchingCubes. Structures and their merged label IDs: MAND=1, CB=2, UAW=3, MAX=4, CV=5, SKIN=6 plus *MASK variants; a SMALL group covers high-def MAND/MAX/RC (root canal). Inference is executed by shelling out to the external nnUNetv2_predict command (subprocess) with -c 3d_fullres -f 0 --disable_tta, NOT by loading the model in-process.

- **Algorithm:** nnU-Net v2, 3d_fullres PlainConvUNet, run via external nnUNetv2_predict subprocess; SimpleITK/ITK + cc3d post-processing; VTK DiscreteMarchingCubes mesh export
- **Entry points:** `AMASSS_CLI/AMASSS_CLI.py`, `AMASSS_CLI/AMASSS_CLI.xml`, `AMASSS/AMASSS.py`

### BATCHDENTALSEG / DentalSegmentator (batch nnU-Net v2 in Slicer)
Slicer Qt widget that batch-processes a folder of NIfTI/GIPL volumes through nnU-Net v2 in-process via SlicerNNUNetLib (Parameter + SegmentationLogic, folds='0', device cuda/cpu/mps). Four selectable models with different label sets: DentalSegmentator & PediatricDentalsegmentator (5 labels: Upper Skull, Mandible, Upper/Lower Teeth, Mandibular canal), NasoMaxillaDentSeg (6 labels, separate Maxilla), UniversalLabDentalsegmentator (55 labels = full FDI per-tooth incl. baby teeth + Mandible/Maxilla/Canal). Produces a Slicer segmentation node, assigns per-label colors/opacities, exports manually built uint16 labelmap to NIfTI and supports STL/OBJ/glTF/VTK(per-label)/VTK(merged) export with windowed-sinc smoothing + quadric decimation. Includes onResolveMirroring(), which rebuilds an FDI labelmap and corrects left/right-mirrored tooth labels using a sagittal mirror plane derived from the central-incisor centroids.

- **Algorithm:** nnU-Net v2 (3d_fullres) run in-process through SlicerNNUNetLib; VTK marching-cubes/threshold/decimation mesh export; numpy left/right mirror correction via incisor-centroid plane
- **Entry points:** `BATCHDENTALSEG/BATCHDENTALSEG.py`, `BATCHDENTALSEG/BATCHDENTALSEGLib/SegmentationWidget.py`, `BATCHDENTALSEG/BATCHDENTALSEGLib/PythonDependencyChecker.py`, `BATCHDENTALSEG/Resources/ML/Dataset111_453CT/nnUNetTrainer__nnUNetPlans__3d_fullres/plans.json`, `BATCHDENTALSEG/Resources/ML/Dataset111_453CT/nnUNetTrainer__nnUNetPlans__3d_fullres/dataset.json`

### CLIC (Mask-R-CNN impacted-canine segmentation)
Slicer module (CLIC.py) that sets up a conda env and runs runner/clic_runner.py to segment impacted-canine bone coverage. The runner builds a torchvision maskrcnn_resnet50_fpn with custom FastRCNNPredictor/MaskRCNNPredictor heads (num_classes=4), loads a *.pth state_dict, then runs 2D slice-wise inference over the volume's Z axis. Each axial slice is min-max normalized to [0,1], stacked to 3 channels, passed to the model; detections with score>=0.7 and mask prob>0.5 paint their label (1=Buccal, 2=Bicortical, 3=Palatal) into the 3D int16 output. Result saved as NIfTI preserving the input nibabel affine/header. This is the only segmenter here that uses an instance-segmentation detection net rather than nnU-Net.

- **Algorithm:** Mask R-CNN (torchvision maskrcnn_resnet50_fpn) 2D slice-wise instance segmentation, .pth state_dict, nibabel I/O
- **Entry points:** `CLIC/CLIC.py`, `CLIC/runner/clic_runner.py`, `CLIC/testing/test_CanineSegmentation.py`

### CNE / CNE_CLI (clinical-notes extraction — NOT image segmentation, out of scope)
Despite living alongside the segmentation modules, CNE does no image processing. CNE_CLI.py loads a local GGUF model with llama-cpp-python (Llama, n_gpu_layers=-1) and extracts structured fields from .txt clinical notes (Ortho or TMJ context, different n_ctx), parsing the LLM's JSON output and writing per-file text extractions. Listed here only to clarify it is a text-LLM tool, not a CBCT segmenter; no reusable segmentation code.

- **Algorithm:** LLM text extraction via llama-cpp-python (GGUF), prompt-based JSON field extraction
- **Entry points:** `CNE/CNE.py`, `CNE_CLI/CNE_CLI.py`, `CNE_CLI/CNE_CLI.xml`

## Reusable utilities

- `AMASSS_CLI/AMASSS_CLI.py :: CleanArray(seg_arr, radius)` — Self-contained mask post-processing: SimpleITK BinaryDilate -> BinaryFillhole -> BinaryErode (morphological close) then cc3d.connected_components to keep only the largest connected component. High-value reusable cleanup for any binary segmentation. Pure SimpleITK+cc3d, no Slicer dependency.
- `AMASSS_CLI/AMASSS_CLI.py :: CropSkin(skin_seg_arr, thickness)` — Extracts a thin skin shell: fill holes, erode by thickness, XOR to get the surface band, keep largest component. Reusable for surface/skin extraction. Pure SimpleITK+numpy+cc3d.
- `AMASSS_CLI/AMASSS_CLI.py :: SetSpacingFromRef(filepath, refFile, interpolator)` — ITK ResampleImageFilter that resamples a labelmap to a reference image's spacing/size/origin/direction using NearestNeighbor (or Linear) interpolation, then copies geometry back via SimpleITK. Core resample/transform math for aligning predictions to original-resolution scans. Reusable.
- `AMASSS_CLI/AMASSS_CLI.py :: SavePrediction(img, ref_filepath, outpath, output_spacing)` — Writes a numpy label array as an int16 image inheriting spacing/direction/origin from a reference scan (SimpleITK). Minimal, dependency-light geometry-preserving label export.
- `AMASSS_CLI/AMASSS_CLI.py :: SavePredToVTK(...) / mesh_from_nrrd` — Converts a label NRRD/NIfTI to colored VTK surface meshes: vtkDiscreteMarchingCubes + vtkSmoothPolyDataFilter, per-label color assignment, plus a merged-append mode. Reusable VTK mesh-extraction recipe (port candidate to vtk.js).
- `BATCHDENTALSEG/BATCHDENTALSEGLib/SegmentationWidget.py :: _exportMergedVTK / _exportVTKPerLabel` — Robust labelmap->mesh export: vtkDiscreteMarchingCubes, vtkWindowedSincPolyDataFilter smoothing (60 iters, passband 0.05), cell-normals, vtkQuadricDecimation(0.4), per-label thresholding, and RAS->LPS transform before writing binary VTK. Strong reference for mesh generation/decimation and coordinate-system handling (port candidate).
- `BATCHDENTALSEG/BATCHDENTALSEGLib/SegmentationWidget.py :: onResolveMirroring + _get_active_label_map` — FDI tooth label dictionaries (5/6/55 labels) and a left/right mirror-correction algorithm using a sagittal mirror plane from central-incisor centroids. Reusable tooth-numbering tables and mirror-geometry logic (the label maps are pure data, easy to port).
- `CLIC/runner/clic_runner.py :: _norm(x2d) and _blank_model(nc)` — _norm: per-slice min-max normalization to [0,1] with epsilon guard. _blank_model: constructs a torchvision Mask R-CNN with custom box/mask predictor heads for nc classes. Compact, self-contained preprocessing + model-builder; _norm is a clean reusable normalization helper.
- `BATCHDENTALSEG/Resources/ML/Dataset111_453CT/.../plans.json` — nnU-Net v2 preprocessing spec to replicate in a web port: CTNormalization, target spacing [0.449,0.312,0.449] (3d_fullres), patch [128,160,112], transpose_forward [1,0,2], and foreground intensity stats (mean 1273.7, std 558.5, clip percentiles 0.5/99.5 = -110/3067) used for CT normalization ((x-mean)/std after clipping).

- **AMASSS_CLI multi-structure nnU-Net (MAND/MAX/CB/UAW/CV/SKIN)** — nnU-Net v2 3d_fullres PlainConvUNet segmenting skull/jaw structures, run via external nnUNetv2_predict subprocess; per-structure binary mask + optional merged labelmap by fixed priority. _(depends on: nnU-Net ONNX export pipeline, CTNormalization preprocessing (nnU-Net plans.json), Volume resampling to target spacing (web))_ Highest-value gap: CBCTer only segments teeth today; mandible/maxilla/cranial-base/airway would be a major feature. The model + harness are portable (proven by the existing tooth UNet), but nnU-Net 3d_fullres at full patch size is heavy for WASM and weights are large multi-structure downloads not on disk. Realistic as web-with-effort for one or two structures; a server-side onnxruntime/Triton microservice is the pragmatic alternative for all structures at full res. Per-structure CleanArray post-proc is nearly free (reuse keepLargestMaskComponent + add morphological close).
- **DentalSegmentator / BATCHDENTALSEG nnU-Net (jaw + teeth + canal, 5/6/55 labels)** — nnU-Net v2 3d_fullres run in-process via SlicerNNUNetLib; produces a multi-label segmentation (Upper Skull, Mandible, Upper/Lower Teeth, Mandibular canal; up to 55 FDI per-tooth labels) with mesh export and mirror correction. _(depends on: nnU-Net ONNX export pipeline, Volume resampling to target spacing (web), FDI tooth label tables + mirror correction (TS))_ Arguably the single best target: one model gives jaw + per-tooth + mandibular canal in one pass, which would replace and far exceed the current single-class tooth UNet + watershed heuristic (no more over-segmentation guessing). The 55-label UniversalLab variant gives true FDI numbering. Effort is XL mainly due to model size and full-res 3d patch inference cost in WASM; consider WebGPU (onnxruntime-web webgpu EP) to make it tractable. CTNormalization stats are pure data, trivial to port.
- **Volume resampling to target spacing (SetSpacingFromRef)** — ITK ResampleImageFilter resampling a volume/labelmap to a reference spacing/size/origin/direction (NearestNeighbor for labels, Linear for intensity), then copying geometry back. Prerequisite for every nnU-Net model port — without spacing normalization the ONNX models produce garbage. Pure math, no Slicer/ITK runtime needed; the app already manipulates Int16Array volumes with [x,y,z] spacing. High leverage, modest effort.
- **nnU-Net CTNormalization preprocessing (plans.json spec)** — Clip foreground intensities to fixed percentile bounds (-110..3067), then (x-mean)/std with dataset stats (mean 1273.7, std 558.5) — the exact normalization nnU-Net models expect, distinct from CBCTer's current per-crop z-score. Cheap and mandatory for nnU-Net parity. The constants are already on disk in BATCHDENTALSEG/Resources/ML/.../plans.json. Without this the ported weights are unusable, so it is a high-value enabler even though it is trivial code.
- **nnU-Net ONNX export pipeline (offline tooling)** — An offline Python script that loads each nnU-Net checkpoint_final.pth into the PlainConvUNet architecture from plans.json and exports a fixed-patch ONNX graph, mirroring the existing scripts/export_tooth_onnx.py + parity validation. This is the keystone enabler for both AMASSS and DentalSegmentator ports. CBCTer already has a working precedent (tooth UNet exported, ~1.4e-5 parity, Dice 0.996). nnU-Net's dynamic architecture makes export fiddlier than a static MONAI UNet, and weights are runtime downloads (not in repo), so budget L. Output models are large (10s-100s of MB) — note download/caching UX.
- **FDI tooth-label tables + left/right mirror correction (onResolveMirroring)** — Pure-data FDI label dictionaries (5/6/55-label sets) plus a geometric algorithm that builds a sagittal mirror plane from central-incisor centroids and corrects mirrored tooth labels. _(depends on: DentalSegmentator / BATCHDENTALSEG nnU-Net (jaw + teeth + canal, 5/6/55 labels))_ Only meaningful once the 55-label UniversalLab model is ported (gives real FDI numbering, a feature CBCTer lacks). Label maps are trivial data; the mirror geometry is a few dozen lines of vector math with no heavy deps. Medium value, blocked on the model.
- **Morphological close radius step (complete CleanArray parity)** — The dilate -> fill-holes -> erode close that precedes keep-largest-component in AMASSS CleanArray; CBCTer has fill-holes and keep-largest but no radius-controlled binary close/open. Small, self-contained, and generally useful for any segmentation cleanup (also improves the current tooth masks and surface generation). The EDT needed already exists in watershed.ts, so dilation/erosion by radius is nearly free. Rounds out CleanArray parity.
- **CropSkin thin-shell / skin-surface extraction** — From a skin segmentation: fill holes, erode by a thickness, XOR to get a surface band, keep-largest — yields a thin skin shell for surface rendering. _(depends on: Morphological close radius step (complete CleanArray parity), AMASSS_CLI multi-structure nnU-Net (MAND/MAX/CB/UAW/CV/SKIN))_ Only relevant if the SKIN structure model is ported, and skin-shell rendering is a niche CBCT use case for a dental viewer. Trivial once erode-by-radius exists, but low priority.
- **Geometry-preserving label export (SavePrediction / SavePredToVTK)** — Write a predicted label array as int16 inheriting spacing/origin/direction from a reference scan (NIfTI/NRRD), and export colored per-label VTK meshes. CBCTer can already display and STL-export masks; the gap is a downloadable NIfTI/NRRD labelmap with correct geometry for interoperability with Slicer/clinical tools. Self-contained writer, no heavy deps. Medium value for clinical workflows.
- **CLIC impacted-canine Mask R-CNN (2D slice-wise instance seg)** — torchvision maskrcnn_resnet50_fpn (num_classes=4) run slice-by-slice over Z; min-max normalize each axial slice to [0,1], 3-channel stack, score>=0.7, paint Buccal/Bicortical/Palatal labels into a 3D map. Niche clinical task (impacted-canine bone coverage), instance-detection net that is the worst fit for WASM, and weights are a runtime download to ~/Documents. Not worth an in-browser port; flag as server-side-only if ever needed. _norm helper is the sole direct-web crumb and is already subsumed by existing normalization code.
- **CNE / CNE_CLI clinical-notes LLM extraction** — GGUF LLM (llama-cpp-python) that extracts structured fields from clinical text notes. Not image segmentation. Explicitly out of segmentation scope; included only for completeness. No reusable segmentation code, no relevance to CBCTer's imaging goals.
