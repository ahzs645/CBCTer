# SADT — MRI2CBCT / MedX / DOCShapeAXI / VFACE / Anonymizer

_Area key: `SADT-misc`_

## Purpose
A cluster of five independent 3D Slicer extensions from SlicerAutomatedDentalTools (SADT), each split into a Slicer GUI module (`X/`) and one or more CLI/worker scripts (`X_CLI/`). They cover: MRI->CBCT cross-modality affine REGISTRATION (not GAN synthesis) plus nnU-Net TMJ cropping; an LLM (BART/Qwen) clinical-note summarization dashboard (MedX); 3D shape classification/regression with explainability on dental/airway/condyle/cleft meshes (DOCShapeAXI, wrapping the external `shapeaxi` package); facial-asymmetry classification from landmark measurements via LightGBM (VFACE) plus nnU-Net dental segmentation; and a text/document PII anonymizer built on Microsoft Presidio (Medical_Data_Anonymizer_Module). Important: despite its name, the "Medical Data Anonymizer" anonymizes free-text documents (docx/pdf/txt/csv/xml/odt) via NLP entity detection — it does NOT touch DICOM headers or pixel data. No model weights ship on disk; all are downloaded at runtime from GitHub releases or Hugging Face.

**Tech stack:** Python 3, 3D Slicer (ScriptedLoadableModule, ctk, qt, MRML), SimpleITK, nibabel (incl. nibabel.processing.resample_from_to), NumPy / SciPy (ndimage.label), PyTorch, torchreg (AffineRegistration), nnU-Netv2 (nnUNetResEncUNetXLPlans, ResEncUNetXL, 3d_fullres) via SlicerNNUNetLib, Hugging Face transformers (BART / Qwen), PyMuPDF (fitz), python-docx, pdfplumber, odfpy, reportlab, lxml, Microsoft Presidio (presidio-analyzer, presidio-anonymizer) + spaCy en_core_web_lg + langdetect, shapeaxi + PyTorch3D + captum (LayerGradCam) + OpenCV, LightGBM + joblib + pandas + openpyxl, VTK (DiscreteMarchingCubes, WindowedSincPolyDataFilter, QuadricDecimation), SlicerConda / WSL2 (DOCShapeAXI desktop env)

**License:** 3D Slicer BSD-style license (repo root file 'LICENCE')

**I/O formats:** NIfTI (.nii/.nii.gz) — MRI2CBCT, VFACE segmentation, GIPL (.gipl/.gipl.gz) — VFACE segmentation input, NRRD/MHD/DICOM(.dcm)/PNG/JPG — accepted by MRI2CBCT resample globbing, ITK transforms (.tfm) — MRI2CBCT registration output, VTK PolyData (.vtk) — DOCShapeAXI input meshes & GradCAM output; VFACE mesh export, STL / OBJ / GLTF — VFACE segmentation mesh export, Labelmap NIfTI — VFACE segmentation output, Excel (.xlsx) — VFACE asymmetry I/O, PDF / DOCX / TXT / CSV / XML / ODT — Anonymizer & MedX document I/O, JSON — nnU-Net dataset.json/plans.json, model_path.json, download_info.json, model checkpoints: .ckpt (DOCShapeAXI / shapeaxi Lightning), .pth (nnU-Net checkpoint_final.pth), LightGBM .txt (joblib)

## Modules

### Medical_Data_Anonymizer_Module (TEXT/document PII anonymizer — web-portable candidate, with caveat)
Single-file Slicer scripted module (767 lines, no separate CLI). Walks an input directory, extracts plain text from .docx/.txt/.pdf/.csv/.xml/.odt, runs Microsoft Presidio (AnalyzerEngine + spaCy en_core_web_lg NER) to detect PII entities, then anonymizes via one of four operators (replace-with-label / redact / hash / mask) and writes anonymized copies plus a file_mappings.csv (original name, anonymized name, UUID). User picks entity types via checkboxes (PERSON, PHONE_NUMBER, EMAIL_ADDRESS, DATE_TIME, LOCATION, US_SSN, MEDICAL_LICENSE, US_DRIVER_LICENSE, CREDIT_CARD, US_BANK_NUMBER, IP_ADDRESS, URL) and a confidence-threshold slider (default 0.5). CAVEAT FOR THE WEB-APP GOAL: this is a server-side Python NLP pipeline (Presidio + spaCy large model, ~hundreds of MB), English-only ('en' hardcoded), and is NOT a DICOM anonymizer — it never reads DICOM tags or burned-in pixel text. Portable to a web app only as a backend microservice (Python/FastAPI calling Presidio), or conceptually re-implemented client-side with a JS NER stack. The clean parts to reuse directly: the multi-format text-extraction logic (extract_text_from_xml recursion; pdfplumber table fallback; docx/odt/csv readers) and the four operator-config mappings. The detection itself is not reusable in-browser without Presidio/spaCy.

- **Algorithm:** Microsoft Presidio (presidio-analyzer + presidio-anonymizer) over spaCy en_core_web_lg NER; operators replace/redact/hash/mask; regex-based recognizers for SSN/credit-card/etc.
- **Entry points:** `Medical_Data_Anonymizer_Module/Medical_Data_Anonymizer_Module.py`, `Medical_Data_Anonymizer_Module/Medical_Data_Anonymizer_Module.py: onAnonymizeButton()`, `Medical_Data_Anonymizer_Module/Medical_Data_Anonymizer_Module.py: anonymize_text_presidio()`, `Medical_Data_Anonymizer_Module/Medical_Data_Anonymizer_Module.py: extract_text_from_xml(), save_str_pdf()`

### MRI2CBCT / MRI2CBCT_CLI (MRI<->CBCT registration + preprocessing + nnU-Net TMJ crop)
Pipeline to spatially align MRI to CBCT for the temporomandibular joint. Despite the name it does NOT synthesize a CBCT from MRI with a GAN; it performs intensity-based AFFINE registration. The 'approximate' step pre-aligns via affine math then optimizes a torchreg AffineRegistration (translation+rotation) by minimizing a custom differentiable Normalized Mutual Information (NMI) loss, sweeping a learning-rate/sigma ParameterGrid for the best transform, and writes an ITK .tfm plus the resampled MRI. Separate utilities do percentile contrast normalization, SimpleITK resampling, left/right and TMJ cropping. TMJ cropping uses an nnU-Netv2 model (Dataset001_myseg, nnUNetResEncUNetXLPlans, 3d_fullres) invoked via the nnUNetv2_predict CLI; AREG_MRI.py wraps a separate ANTs-style/AREG registration. Final segmentation/registration relies on the SlicerNNUNet extension.

- **Algorithm:** Intensity-based affine registration (torchreg AffineRegistration + differentiable NMI loss, De Vos 2020) with affine pre-alignment via SVD-orthogonalized rotation matching; SimpleITK Resample; nnU-Netv2 (ResEncUNetXL) for TMJ ROI segmentation/crop.
- **Entry points:** `MRI2CBCT_CLI/MRI2CBCT_CLI_utils/approximate.py: approximation()`, `MRI2CBCT_CLI/MRI2CBCT_CLI_utils/approx_utils.py`, `MRI2CBCT_CLI/MRI2CBCT_CLI_utils/nmi.py: NMI`, `MRI2CBCT_CLI/MRI2CBCT_CLI_utils/resample.py`, `MRI2CBCT_CLI/MRI2CBCT_CLI_utils/normalize_percentile.py`, `MRI2CBCT_CLI/MRI2CBCT_TMJ_CROP/MRI2CBCT_TMJ_CROP.py`, `MRI2CBCT_CLI/MRI2CBCT_CLI_utils/AREG_MRI.py`, `MRI2CBCT/MRI2CBCT.py`

### MedX / MedX_CLI (clinical-note LLM summarization + dashboard)
Ingests clinical notes (PDF via PyMuPDF/fitz, Word via python-docx), cleans and chunks text along section headers (CLINICAL/RADIOGRAPHIC EXAMINATION), and runs a Hugging Face seq2seq model to extract structured key-value pairs / summaries. MedX_Summarize.py loads a BART (BartForConditionalGeneration) model from a local path via transformers and generates with beam search; comments reference a fine-tuned Qwen1.5B variant. A second CLI (MedX_Dashboard) aggregates per-patient predictions into a dashboard/CSV (dashboard_full_dataframe.csv example included). Pure NLP/LLM, no imaging.

- **Algorithm:** Transformers seq2seq generation (BART / Qwen) with GenerationConfig (beams=4, no_repeat_ngram, sampling); regex section-aware paragraph chunking; key:value regex extraction.
- **Entry points:** `MedX_CLI/MedX_Summarize/MedX_Summarize.py`, `MedX_CLI/MedX_CLI_utils/utils.py`, `MedX_CLI/MedX_Dashboard/MedX_Dashboard.py`, `MedX/MedX.py`

### DOCShapeAXI / DOCShapeAXI_CLI (3D mesh classification + GradCAM explainability)
Classifies/regresses dental & craniofacial VTK surface meshes for three tasks — Nasopharynx Airway Obstruction, Mandibular Condyles, Alveolar Bone Defect in Cleft — each as binary/severity/regression. Wraps the external `shapeaxi` package: builds a SaxiDataset, loads a Lightning checkpoint (saxi_nets_lightning, e.g. SaxiMHAFBClassification — a multi-head-attention model that renders multiple 2D views of the mesh and samples point clouds), predicts, then produces per-vertex GradCAM explainability via captum LayerGradCam on the convnet blocks, writing colored VTK surfaces. The Slicer module runs the CLI inside a conda/WSL environment (SlicerConda) and installs PyTorch3D. Heavy desktop/GPU stack — poor web-port candidate.

- **Algorithm:** Multi-view rendering + point-cloud + multi-head-attention classifier (shapeaxi SaxiMHAFB*) on PyTorch3D; captum LayerGradCam explainability; checkpoints (.ckpt) downloaded from GitHub releases.
- **Entry points:** `DOCShapeAXI_CLI/DOCShapeAXI_CLI.py: saxi_predict(), saxi_gradcam()`, `DOCShapeAXI_CLI/model_path.json`, `DOCShapeAXI/DOCShapeAXI.py`, `DOCShapeAXI/DOCShapeAXI_utils/install_pytorch.py`

### VFACE / VFACE_CLI (facial asymmetry classification + nnU-Net dental segmentation)
Two distinct capabilities. (1) VFACE_CLI.py: facial-asymmetry classification from an Excel sheet of landmark-derived measurements using three pre-trained LightGBM models loaded via joblib (sym_asymm.txt, mand_asym.txt, max_asym.txt) — first predicts symmetric/asymmetric, then sub-classifies mandible vs maxilla involvement; outputs an annotated Excel. (2) VFACE_utils/segmentation_logic.py: a self-contained dental CBCT/CT segmentation engine over nnU-Net (via SlicerNNUNetLib) supporting four downloadable models (DentalSegmentator, PediatricDentalsegmentator, NasoMaxillaDentSeg, UniversalLabDentalsegmentator), with rich VTK mesh post-processing/export (marching cubes, windowed-sinc smoothing, quadric decimation, per-label & merged VTK/STL/OBJ/NIfTI, RAS->LPS). VFACE_utils also has landmark geometry classes (Point/Line/Measure: distances, angles, vector projection/rejection).

- **Algorithm:** LightGBM (joblib) tabular classifiers for asymmetry; nnU-Netv2 3d_fullres dental segmentation; VTK marching-cubes + windowed-sinc smoothing + quadric decimation mesh extraction; NumPy landmark geometry (distances/angles/projections).
- **Entry points:** `VFACE_CLI/VFACE_CLI.py: load_models(), classify_symmetry()`, `VFACE/VFACE_utils/segmentation_logic.py`, `VFACE/VFACE_utils/Measure.py`, `VFACE/VFACE_utils/Point.py`, `VFACE/VFACE_utils/Line.py`, `VFACE/Resources/ML/Dataset111_453CT/.../plans.json`, `VFACE/Resources/ML/download_info.json`

## Reusable utilities

- `MRI2CBCT_CLI/MRI2CBCT_CLI_utils/nmi.py` — Self-contained differentiable Normalized Mutual Information loss (nmi_gauss, nmi_gauss_mask, NMI(_Loss)) using Gaussian Parzen-window histograms — reusable for any intensity-based registration/similarity, torch-only.
- `MRI2CBCT_CLI/MRI2CBCT_CLI_utils/approx_utils.py` — High-value IO/transform math: nib<->sitk conversion (nib_to_sitk/sitk_to_nib), SVD orthogonalization of direction matrices, RAS<->LPS transform conversion (convert_transform_for_slicer), SVD-based affine pre-alignment (prealign_mri_to_cbct), resample_to_match, downsample (torch trilinear). Pure NumPy/nibabel/SimpleITK — portable.
- `MRI2CBCT_CLI/MRI2CBCT_CLI_utils/resample.py` — General SimpleITK Resample/resample_fn with fit/iso-spacing, centering, reference-image matching, Z-flip mirroring, and CSV/dir/single-file batch driver — standalone CLI, no Slicer dependency.
- `MRI2CBCT_CLI/MRI2CBCT_CLI_utils/normalize_percentile.py` — Percentile-based contrast normalization for NIfTI volumes (compute_thresholds, enhance_contrast, normalize) — clean, dependency-light (SimpleITK+NumPy).
- `MRI2CBCT_CLI/MRI2CBCT_TMJ_CROP/MRI2CBCT_TMJ_CROP.py` — Geometry-preserving NIfTI cropping helpers: crop_with_affine (updates affine origin) and biggest_cc (largest connected component via scipy.ndimage.label) — reusable mask/ROI utilities.
- `MedX_CLI/MedX_CLI_utils/utils.py` — Document text extraction + section-aware chunking: extract_text_from_pdf (PyMuPDF), extract_text_from_word (python-docx), clean_text, split_text_by_paragraphs, create_chunks_from_paragraphs, extract_key_value_pairs — reusable NLP preprocessing independent of the LLM.
- `Medical_Data_Anonymizer_Module/Medical_Data_Anonymizer_Module.py` — Multi-format text extractors (docx/txt/pdf-with-table-fallback/csv/xml-recursive/odt) and Presidio operator-config builder (replace/redact/hash/mask) — the extraction layer is reusable; detection requires Presidio backend.
- `VFACE/VFACE_utils/segmentation_logic.py` — Standalone VTK mesh post-processing pipeline from a labelmap: marching cubes, windowed-sinc smoothing, quadric decimation, per-label and merged VTK/STL/OBJ export with RAS->LPS transform — strong reference for client/server mesh generation (though it calls slicer APIs for IO).
- `VFACE/VFACE_utils/Measure.py` — NumPy landmark geometry: point-to-point distance, point-to-line/angle computation, vector projection/rejection onto axes — portable analytic math for cephalometric-style measurements.
- `VFACE_CLI/VFACE_CLI.py` — clean_name() robust column-name sanitizer and the joblib LightGBM load+hierarchical-predict pattern (feature_name_-driven column selection) — reusable tabular-ML inference skeleton.

The NMI loss and SVD pre-alignment are the salvageable algorithmic nuggets, but a full optimizing registration in-browser is a research project. Revisit only if multimodal TMJ registration becomes a product goal; better as a server-side service.
- **MRI2CBCT / VFACE nnU-Netv2 segmentation runtime (ResEncUNetXL, 3d_fullres, SlicerNNUNetLib)** — Heavy 3D nnU-Net inference for TMJ ROI cropping and dental/maxillofacial segmentation, run via the nnUNetv2_predict CLI inside conda/WSL with downloaded .pth weights. Medium value (better dental/TMJ segmentation than the current tooth UNet) but the ResEnc-XL nnU-Net itself is desktop/GPU. - **DOCShapeAXI 3D mesh classification + GradCAM (shapeaxi SaxiMHAFB, PyTorch3D, captum)** — Multi-view-render + point-cloud multi-head-attention classifier over VTK surface meshes with per-vertex GradCAM explainability; runs in conda/WSL with PyTorch3D. Poor web-port candidate and tangential to CBCTer's CBCT-viewer scope (airway/condyle/cleft shape classification). Server-side only if ever needed.
- **MedX clinical-note LLM summarization (BART/Qwen seq2seq) + document chunking utils** — PDF/Word ingestion, section-aware chunking, and HF seq2seq generation to produce structured summaries of clinical notes; plus reusable text extraction/chunking utilities. Out of scope for an imaging viewer (pure NLP, no imaging). Only the lightweight document-parsing helpers are reusable; the BART/Qwen model is server-side. Not worth porting for CBCTer.
- **VFACE LightGBM tabular asymmetry classifiers (sym/mand/max) + joblib hierarchical-predict skeleton** — Three pre-trained LightGBM boosters predicting facial symmetry then mandible/maxilla involvement from landmark-derived Excel measurements; plus a clean column-sanitizer and feature_name_-driven predict pattern. _(depends on: VFACE Measure.py / Point.py / Line.py landmark geometry (distance, point-to-line, angle, vector projection/rejection))_ Technically very portable (small models, no GPU) but low value unless facial-asymmetry analysis from landmarks is a target use case, which is outside CBCTer's current CBCT-segmentation/viewer focus. Cheap to do if the landmark workflow exists; otherwise skip.

## Already covered by CBCTer (skip)

- VFACE segmentation_logic.py mesh post-processing (marching cubes + windowed-sinc smoothing + quadric decimation + per-label/merged STL/OBJ/VTK export): CBCTer already has src/lib/segmentation/maskMesh.ts (marching-tetrahedra iso extraction + voxel surface, Laplacian smoothing, index-based decimation, binary STL + ASCII PLY) and src/lib/surface/generateSurface.ts (worker driver with area/volume/triangle metrics). Re-porting the VTK pipeline adds little beyond OBJ output and windowed-sinc (vs Laplacian) smoothing.
